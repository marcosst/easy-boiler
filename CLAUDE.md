# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Setup & Development

```bash
make setup    # Sync deps with uv, copy .env.example → .env, create data/
make dev      # Run migrations + start uvicorn with hot-reload on :8000
```

Requirements: Python 3.12+, `uv`, and `dbmate` on PATH (`dbmate` is installed automatically in Docker).

## Database Migrations

Migrations are managed with [dbmate](https://github.com/amacneil/dbmate). Files live in `db/migrations/`.

```bash
make db-new name=create_users   # Create a new migration file
make migrate                    # Run pending migrations
make db-rollback                # Undo last migration
```

Each migration file has `-- migrate:up` and `-- migrate:down` sections. `db/schema.sql` is auto-generated by dbmate — do not edit it manually.

## Architecture

**Stack:** FastAPI + Jinja2 templates + HTMX + Tailwind + SQLite via aiosqlite, managed with `uv`.

- `app/main.py` — FastAPI app, route definitions. Full-page routes return Jinja2 `TemplateResponse`; HTMX partial routes live under `/htmx/` and return HTML fragments.
- `app/database.py` — async SQLite connection via `aiosqlite`. Use `get_db()` as a FastAPI dependency (`Depends(get_db)`). DB path is resolved from `DATABASE_URL` env var.
- `app/templates/` — Jinja2 templates. `base.html` is the base layout; `partials/` contains HTMX fragment templates.
- `static/` — Static assets (CSS, JS) mounted at `/static`.
- `db/migrations/` — dbmate SQL migration files.

The pattern for new features: add a route in `main.py`, inject `db = Depends(get_db)` for DB access, create templates in `app/templates/` (full pages) or `app/templates/partials/` (HTMX fragments), and create a migration if schema changes are needed.
